# 毕设答辩演示顺序说明

## 文档目的

这份文档用于固定当前版本的答辩演示顺序，避免现场演示时在页面之间来回跳转，确保每一步都有明确讲解点。

## 演示前准备

1. 确认真机或模拟器已经安装最新 `debug` 包。
2. 如果需要演示同步，先启动本地后端：

```powershell
./gradlew.bat :backend:bootRun
```

3. 确认 `local.properties` 中的 `base.url` 指向当前电脑可访问地址。
4. 准备一张可用于 OCR 的支付截图或票据图片。

## 建议演示顺序

### 1. 登录进入主页

演示操作：
1. 打开 App。
2. 使用演示账号 `demo / 123456` 登录。

讲解重点：
1. App 支持本地会话保持，重新打开时可自动识别登录状态。
2. 当前版本已经具备完整主页、统计、个人中心和业务入口。

### 2. 展示分类管理

演示操作：
1. 从“我的”页面进入“分类管理”。
2. 展示默认收入/支出分类。
3. 新增一个自定义分类。

讲解重点：
1. 首次进入会自动生成默认分类。
2. 自定义分类支持新增、编辑、删除。
3. 删除分类时，历史交易会自动迁移到默认“未分类”。

### 3. 手动新增一笔交易

演示操作：
1. 从首页进入“新增记账”。
2. 录入金额、分类、日期、商户和备注。
3. 保存交易后返回首页。

讲解重点：
1. 当前支持收入和支出双类型记账。
2. 保存后首页最近记录和月度摘要会立即刷新。

### 4. 编辑或删除一笔交易

演示操作：
1. 在首页最近记录或交易页打开一条已保存交易。
2. 演示编辑回填。
3. 演示删除确认。

讲解重点：
1. 编辑会把原始记录完整回填到表单。
2. 删除采用物理删除，操作反馈直接可见。

### 5. 展示 OCR 带入

演示操作：
1. 从首页进入“票据识别”。
2. 选择图片或拍照。
3. 点击“开始识别”。
4. 展示金额、日期、商户与原始识别文本。
5. 点击“带入记账表单”并保存。

讲解重点：
1. OCR 识别结果支持人工核对后再保存。
2. 当前版本重点保证生活支付截图场景下的金额、日期、商户提取可演示。
3. 最近识别记录会保留在本地，方便重复演示。

### 6. 展示统计页

演示操作：
1. 切换到统计页。
2. 切换日、周、月三个周期。
3. 展示总收入、总支出、结余、饼图和柱状图。

讲解重点：
1. 统计基于真实本地交易数据实时生成。
2. 支出分类占比和周期收支趋势可以直接辅助用户理解消费结构。

### 7. 展示本地同步

演示操作：
1. 进入“我的”页。
2. 点击“立即同步”。
3. 展示同步结果提示。

讲解重点：
1. 当前同步是“演示级本地后端同步”，重点展示前后端联调闭环。
2. 点击后会先上传本地变更，再拉取当前云端快照。
3. 首次同步和后续无改动同步的提示文案已经区分清楚。

### 8. 退出登录

演示操作：
1. 点击“退出登录”。
2. 返回登录页。

讲解重点：
1. 当前会清理本地会话状态。
2. 重新登录后可以继续查看原有本地数据和同步结果。

## 现场讲解建议

1. 把“生活类记账 + OCR 提升录入效率 + 本地同步演示闭环”作为主线，不要把重心放在后端安全。
2. 如果老师先问技术栈，可以按 `Kotlin + MVVM + Hilt + Room + Retrofit + Material Design 3 + Spring Boot 演示后端` 这条线回答。
3. 如果老师追问 OCR，重点说明当前实现优先服务支付截图与票据的演示识别，不强调生产级泛化。
4. 如果老师追问同步，说明当前后端主要用于演示前后端分离和数据互通，不是正式部署版。

## 当前版本不主动展开的内容

1. 不主动强调生产级安全、复杂权限模型或分布式部署。
2. 不主动强调高并发、多角色管理或企业级 OCR 能力。
3. 不主动展开离线冲突合并的复杂策略，除非老师继续追问。
