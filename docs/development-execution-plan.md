# Android OCR 记账 App 开发执行文档

## 1. 文档目的

本文档用于约束后续实际开发顺序，确保开发过程按照固定阶段推进，而不是在多个模块之间来回切换。

后续开发必须遵循以下原则：

1. 每次只推进一个明确阶段。
2. 每个阶段开始前先确认输入条件。
3. 每个阶段结束前必须完成最小可运行闭环。
4. 每个阶段都要有验证方式。
5. 每次提交都必须与当前阶段目标对应，并同步推送到 GitHub。

## 2. 当前项目状态

### 2.1 已完成内容

当前仓库已经完成以下基础骨架能力：

1. Android 工程初始化与 GitHub 同步链路。
2. `MVVM + Hilt + Room + Retrofit + Navigation + DataStore` 基础接入。
3. 单 Activity + Bottom Navigation 主框架。
4. 本地数据库实体、DAO、数据库类骨架。
5. 远程接口 Service、请求模型、响应模型骨架。
6. 首页、统计、我的、分类、OCR、记账录入占位页面。

### 2.2 当前未完成内容

以下内容仍然是业务空壳或未实现状态：

1. 登录注册真实业务流程。
2. App 启动时的登录态判断与页面路由。
3. 分类管理真实增删改查。
4. 交易真实增删改查。
5. 统计聚合与图表。
6. OCR 识别真实流程。
7. 同步队列与远程同步逻辑。
8. 最终 UI 统一收口。

## 3. 总体开发顺序

后续开发固定采用以下顺序：

1. 阶段 A：全局主题与认证壳层收口。
2. 阶段 B：用户登录注册与本地会话保持。
3. 阶段 C：分类管理闭环。
4. 阶段 D：交易增删改查闭环。
5. 阶段 E：统计聚合与可视化。
6. 阶段 F：OCR 识别与记账填充。
7. 阶段 G：远程同步闭环。
8. 阶段 H：联调整理、测试补全与演示优化。

说明：

1. 认证先于分类和交易，因为后者依赖用户身份与会话。
2. 分类先于交易，因为交易录入依赖分类选择。
3. 本地交易先于统计，因为统计依赖真实本地数据。
4. OCR 先做本地识别与填充，再接同步，避免同时处理两条复杂链路。
5. 同步放在后段实现，因为它依赖登录、分类、交易和本地数据结构稳定。

## 4. 每阶段详细规划

## 4.1 阶段 A：全局主题与认证壳层收口

### 目标

将当前项目从“骨架页面集合”收口成“具备登录前后页面边界”的真实应用壳层，并统一全局 Material Design 3 主题基线。

### 输入条件

1. 主 Activity 与导航已可运行。
2. Hilt、Navigation、DataStore 已接入。

### 本阶段任务

1. 将全局配色调整为符合要求的 Material Design 3 生活类风格：
   - 不使用绿色。
   - 不使用天蓝色。
   - 以暖橙、珊瑚、暖米白、中性灰为主。
2. 增加启动路由页或启动判断逻辑。
3. 明确“登录前页面”和“登录后页面”的导航边界。
4. 约束底部导航只在登录后主页面显示。

### 产出物

1. 统一主题色与基础样式。
2. 启动页或启动路由逻辑。
3. 认证页面导航入口。

### 完成标准

1. App 启动后能进入启动判定流程。
2. 未登录不会直接进入主业务页。
3. 登录后才进入主页面容器。

### 验证方式

1. `assembleDebug`
2. 手动运行并验证路由跳转

## 4.2 阶段 B：用户登录注册与本地会话保持

### 目标

完成完整的“注册 -> 登录 -> 保存 token -> 启动自动识别登录状态 -> 退出登录”闭环。

### 输入条件

1. 阶段 A 已完成。
2. 认证相关接口模型已具备骨架。
3. `SessionManager` 已存在。

### 本阶段任务

1. 创建认证模块包结构：
   - `ui/auth`
   - `repository/auth`
   - 必要的 mapper / ui state
2. 实现认证仓库：
   - 登录
   - 注册
   - 退出登录
   - 获取当前会话
3. 实现登录页 UI：
   - 用户名
   - 密码
   - 登录按钮
   - 跳转注册页
4. 实现注册页 UI：
   - 用户名
   - 密码
   - 昵称
   - 邮箱
   - 手机号
   - 注册按钮
5. 实现登录与注册 ViewModel：
   - 输入校验
   - loading 状态
   - 成功跳转
   - 失败提示
6. 登录成功后保存：
   - token
   - userId
   - username
   - nickname
7. 登录成功后拉起主页面。
8. 退出登录时清理本地会话。

### 本阶段不做

1. 找回密码
2. refresh token
3. 复杂账号安全策略

### 产出物

1. 登录页
2. 注册页
3. 启动自动判定登录状态
4. 退出登录能力

### 完成标准

1. 能注册新用户。
2. 能登录成功并进入主页面。
3. 重新打开 App 时能根据本地 token 判断是否直接进入主页。
4. 退出后会回到登录页。

### 验证方式

1. `assembleDebug`
2. 模拟成功登录流程
3. 模拟退出流程
4. 单元测试覆盖输入校验和 session 保存逻辑
