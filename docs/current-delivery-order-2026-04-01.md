## 文档目的

这份文档用于固定当前主线开发顺序，避免在多个模块之间来回跳转，保证每一步都形成可演示、可验证、可提交的最小闭环。

执行原则：
1. 后续开发默认以本文档中的顺序为准。
2. 每完成一个阶段，必须先验证通过，再进入下一个阶段。
3. 每个有意义的阶段完成后，都要提交并推送到 GitHub。

## 当前主线顺序

从现在开始，开发顺序固定为：
1. S1：分类管理与交易增删改查闭环
2. S2：统计页联动验证与轻量收口
3. S3：演示用本地后端同步
4. S4：答辩演示路径优化与全流程联调

## 阶段说明

### S1：分类管理与交易增删改查闭环

目标：
1. 确保默认分类初始化稳定。
2. 确保新增、编辑、删除分类完整可用。
3. 确保删除分类后，历史交易自动迁移到“未分类”。
4. 确保交易新增、编辑、删除形成完整闭环。
5. 确保 OCR 带入后的交易可以正常保存到真实列表。
6. 确保首页最近交易与月度摘要会随交易变动即时刷新。

本阶段检查点：
1. 分类重名校验是否合理，默认分类是否不可删不可改。
2. 交易编辑回填是否完整，包括金额、日期、分类、商户、备注。
3. 空态文案、删除确认、保存反馈是否适合演示。
4. 旧数据或乱码默认分类是否会破坏“未分类”迁移逻辑。

完成标准：
1. 分类页可以稳定切换收入/支出并完成增删改。
2. 交易页可以稳定完成新增、编辑、删除。
3. 首页能看到最近交易和月度摘要联动刷新。
4. 相关单元测试与构建验证通过。

### S2：统计页联动验证与轻量收口

目标：
1. 确认日、周、月切换逻辑稳定。
2. 确认交易数据变化后，统计页图表和摘要会同步刷新。
3. 只修正演示过程中发现的真实问题，不做额外复杂扩展。

完成标准：
1. 统计页在真实交易数据下展示正确。
2. 图表无明显闪烁、空态错误或数据错位。

### S3：演示用本地后端同步

目标：
1. 与本机 Spring Boot 后端打通演示级同步。
2. 提供手动同步入口与结果反馈。
3. 保证离线状态下不影响本地记账。

建议顺序：
1. 建立 SyncRepository 或等价同步调度层。
2. 读取本地待同步队列。
3. 打通分类、交易、用户的最小同步闭环。
4. 在个人中心提供“立即同步”入口。

完成标准：
1. 可完成一次本地到后端的演示级同步。
2. 可将后端数据拉回本地。
3. 同步成功与失败反馈明确。

### S4：答辩演示路径优化与全流程联调

目标：
1. 优化首页、分类、交易、OCR、统计、个人中心之间的演示节奏。
2. 补齐必要提示文案与异常反馈。
3. 确保真机演示路径稳定、顺手、易讲解。

完成标准：
1. 主链路无明显卡顿、错页、重复点击问题。
2. 能按答辩顺序完整演示一次核心业务流程。
